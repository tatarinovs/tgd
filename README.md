# TGD

Мощный загрузчик медиафайлов из Telegram каналов и групп, оптимизированный для работы с большим количеством файлов.

## Основные возможности

- **Умный менеджер закачек**: 
  - Раздельные очереди для «тяжелых» и «легких» файлов. 
  - Тяжелые файлы (видео, документы) скачиваются через `FastTelethonhelper` с использованием нескольких соединений.
  - Легкие файлы (фото, голосовые, мелкие видео) скачиваются параллельно в несколько потоков, не дожидаясь завершения тяжелых закачек.
- **Экстремальная скорость**: 
  - Использование `FastTelethonhelper` для многопоточной загрузки одного файла.
  - **AES-CTR Speedup**: Динамический патч (Monkey-patch) для Telethon, заменяющий медленный Python-шифратор на быстрый OpenSSL (через `cryptography`), что ускоряет работу через MTProxy до 2300 MB/s.
- **Устойчивость**: Автоматические повторные попытки при сбоях, обработка таймаутов и поддержка **2FA (двухфакторной аутентификации)**.
- **Поддержка Прокси (MTProto/SOCKS5/HTTP)**: Встроенная возможность пропускать трафик через прокси для обхода блокировок. Поддерживаются официальные ссылки Telegram (`tg://proxy?...`), включая **FakeTLS MTProxy** (секреты, начинающиеся с `ee`).
- **Прогресс-бары**: Наглядное отображение процесса загрузки для каждого файла с помощью `tqdm`.

## Установка

**Требования:** Python 3.9 или новее.

1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/tatarinovs/tgd.git
   cd tgd
   ```
2. Установите проект и зависимости:
   ```bash
   pip install -e .
   ```
   Для максимальной скорости и поддержки всех типов прокси установите дополнительные зависимости:
   ```bash
   pip install cryptography python-socks[asyncio] FastTelethonhelper
   ```
   Если планируете использовать **FakeTLS MTProxy** (секреты, начинающиеся с `ee`):
   ```bash
   pip install TelethonFakeTLS
   ```

После установки будут доступны команды:
- `tgd` — загрузчик медиафайлов
- `tgv` — проверка целостности файлов

## Настройка

Создайте файл `.env` в корневой директории проекта и добавьте туда свои данные от Telegram API:

```env
APP_API_ID=ваш_api_id
APP_API_HASH=ваш_api_hash
PHONE_NUMBER=+79991234567

# Дополнительные параметры (опционально):
# PROXY=socks5://192.168.1.10:10808
# PROXY=tg://proxy?server=192.168.1.10&port=443&secret=...
# TIMEOUT=3600
# RETRIES=3
# WORKERS=6              # Количество потоков для легких файлов
# HEAVY_WORKERS=1        # Количество потоков для тяжелых файлов
# HEAVY_THRESHOLD=100    # Порог "тяжелого" файла в МБ
# QUEUE_SIZE=50          # Размер окна предпросмотра сообщений
```

### Как получить API ID и Hash:

1. Перейдите на сайт [my.telegram.org](https://my.telegram.org).
2. Авторизуйтесь под своим номером телефона (код придет в само приложение Telegram).
3. Перейдите в раздел **"API development tools"**.
4. Создайте новое приложение (заполните `App title` и `Short name` любыми значениями, например `MyDownloader`).
5. После нажатия "Create application" вы увидите `App api_id` и `App api_hash`. 
6. Скопируйте их и вставьте в соответствующие поля в файле `.env`.

> [!IMPORTANT]
> Никогда не передавайте ваш `api_hash` третьим лицам. Это секретный ключ вашего приложения.

## Использование

Запустите скрипт через терминал:

```bash
python tgd.py [GROUP_ID] [OUTPUT_DIR]
```

### Параметры:
- `group_id`: ID группы или канала: число, ссылка (`https://t.me/...`) или `@username`.
- `output_dir`: Путь к папке, куда будут сохраняться файлы.

### Дополнительные опции (имеют приоритет над `.env`):
- `--env`: Путь к файлу `.env` (по умолчанию `.env`).
- `--timeout`: Тайм-аут загрузки одного файла в секундах (по умолчанию 3600).
- `--retries`: Количество попыток при сбое (по умолчанию 3).
- `--workers`: Потоков для легких файлов (по умолчанию 6).
- `--heavy-workers`: Потоков для тяжелых файлов (по умолчанию 1).
- `--heavy-threshold`: Порог тяжелого файла в МБ (по умолчанию 100).
- `--queue-size`: Размер очереди сообщений (по умолчанию 50). Если в канале очень много тяжелых файлов, увеличьте это значение (например, до 500), чтобы легкие воркеры не простаивали.
- `--proxy`: Прокси-сервер (SOCKS5/HTTP/MTProto).

## Пример

```bash
python tgd.py -1001234567890 downloads/my_channel
```

## Проверка целостности файлов

Для массовой проверки скачанных файлов (не битые ли они и открываются ли) предусмотрен скрипт `verify.py`.

```bash
python verify.py [ПАПКА_С_ФАЙЛАМИ]
```

### Возможности:
- Проверка структуры изображений (`.jpg`, `.png`, `.webp`, `.jpeg`).
- Проверка читаемости видеофайлов (`.mp4`, `.mov`, `.mkv`, `.avi`, `.webm`) путем попытки чтения первого кадра (OpenCV) или глубокого анализа контейнера (FFprobe).
- Рекурсивный поиск файлов во всех поддиректориях.
- Интерактивный ввод параметров, если они не указаны при запуске.
- Автоматическое удаление битых файлов с помощью флага `--delete`.

### Работа с FFprobe

Для более качественной проверки видео рекомендуется использовать `ffprobe` (часть пакета FFmpeg).

**Где скрипт ищет ffprobe:**
1. По пути, указанному явно: `--ffprobe C:\path\to\ffprobe.exe`
2. В папке со скриптом (`ffprobe.exe`).
3. В системном PATH (если FFmpeg установлен в системе).

Если `ffprobe` не найден, скрипт переключится на упрощенную проверку (OpenCV), которая проверяет только возможность открытия файла и чтение первого кадра.

### Пример:
```bash
python verify.py downloads/my_channel --delete
```
